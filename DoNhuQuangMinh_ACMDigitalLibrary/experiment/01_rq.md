### 1. Initial Research Question (RQ)

**Initial RQ:**
"Đối với user stories viết theo format Connextra (P), LLM tự động sinh Gherkin scenarios và step definitions (I) so với BDD scenarios viết thủ công (C) có đạt semantic similarity ≥ 0.85 với expected behavior và executable không lỗi cú pháp không (O)?"

### 2. Initial PICO

* **Population (P):** User stories in Connextra format ("As a... I want... So that...")
* **Intervention (I):** LLM automatically generating Gherkin scenarios and step definitions.
* **Comparison (C):** BDD scenarios manually written by experts.
* **Outcome (O):** Semantic similarity ≥ 0.85 and executable syntax (no parsing errors).

### 3. Adjustments and Sources

Based on the **Evidence Table**, **PRISMA screening process**, and **Gap Analysis**, the following refinements were made to improve clarity, reproducibility, and research validity.

| Element to Adjust          | Refinement / Adjustment                                       | Source / Justification                                                                                                  |
| -------------------------- | ------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------- |
| LLM Selection              | Specify GPT-4o zero-shot                                      | Evidence Table shows GPT-3, GPT-4, Claude, and Llama are used, but GPT-4o has not been directly evaluated for this task |
| Model Configuration        | Set temperature = 0                                           | Reproducibility best practice in Software Engineering experiments                                                       |
| Semantic Similarity Metric | Use cosine similarity based on all-MiniLM-L6-v2 embeddings    | Provides an objective and reproducible semantic evaluation metric                                                       |
| Similarity Threshold       | Set target ≥ 0.85                                             | Adopted from related work or defined as project evaluation threshold                                                    |
| Executable Threshold       | Set executable syntax rate ≥ 80%                              | No standard threshold identified in literature; project-defined acceptance criterion                                    |
| Dataset Scope              | User stories collected from ≥ 3 Software Engineering projects | Improves generalizability and reduces domain-specific bias                                                              |

### 4. Final Adjusted PICO

* **P:** User stories in Connextra format sourced from **at least three Software Engineering projects**.
* **I:** **GPT-4o zero-shot (temperature = 0)** generating Gherkin acceptance test scenarios.
* **C:** Gherkin scenarios manually written by **expert software testers** for the same user stories.
* **O:**

  * **Metric 1:** Cosine similarity (all-MiniLM-L6-v2) ≥ 0.85
  * **Metric 2:** Executable syntax rate ≥ 80%

### 5. Final Research Question

"Does **GPT-4o zero-shot (temperature = 0)** generate Gherkin acceptance test scenarios that achieve **cosine semantic similarity ≥ 0.85** with expert-written scenarios **AND** an **executable syntax rate ≥ 80%** when applied to Connextra-format user stories from Software Engineering projects?"
